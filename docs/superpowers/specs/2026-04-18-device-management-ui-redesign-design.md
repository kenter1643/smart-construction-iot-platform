# 设备管理页 UI 重设计方案

## 背景

当前设备管理界面更像演示页面，视觉元素偏散，后台系统的导航、信息层级、操作密度和状态反馈不够成熟。用户明确要求重新设计交互界面和 UI，并按通用后台的做法进行设计。

本次只重做现有设备管理页面，不扩展到完整多页面后台、监控总览、视频中心或告警中心。

## 设计目标

- 让页面呈现为成熟的通用后台，而不是演示界面。
- 保留智慧工地的监控运营气质，避免纯白表格页过于普通。
- 优先支持设备列表查看、筛选、状态判断、创建、编辑、删除等当前已有功能。
- 在不改后端接口的前提下提升页面结构、视觉一致性和交互反馈。
- 保持单文件静态前端可运行，避免为本次 UI 重设计引入复杂构建链路。

## 非目标

- 不实现新的后端 API。
- 不做完整后台路由系统。
- 不做工地地图点位、大屏动效或视频监控页面。
- 不引入 React、Vue 或其他前端框架。
- 不修改设备管理服务的数据模型。

## 设计方向

采用“混合型运营后台”风格：

- 顶部栏和侧边导航使用深色底，建立监控指挥中心的专业感。
- 主内容区使用浅色背景、白色卡片和高对比文字，保证表格、筛选和表单的可读性。
- 强调通用后台标准结构：左侧导航、顶部项目栏、页面标题、统计卡片、筛选工具条、数据表格、详情侧栏。
- 避免 emoji 作为图标，使用内联 SVG 图标，保持专业后台视觉。

## 页面结构

页面采用固定后台壳布局：

1. 顶部栏

- 展示系统名称“智慧工地运营中心”。
- 展示项目上下文，例如“城南工地 A”。
- 提供全局搜索、通知入口和用户身份展示。
- 使用深色背景，与侧边导航形成统一框架。

2. 侧边导航

- 包含总览看板、设备管理、视频监控、告警中心、数据分析、系统设置。
- 当前页面高亮“设备管理”。
- 底部显示 API 连接状态和最近同步时间。
- 在移动端收敛为顶部区域下方的横向导航或折叠菜单。

3. 页面头部

- 标题为“设备管理”。
- 副标题说明该页面用于统一管理传感器、摄像头、控制器与执行器。
- 主操作按钮为“新增设备”。
- 次要操作保留“刷新”或“导出”。

4. 状态统计卡片

展示 4 个核心指标：

- 总设备数。
- 在线设备数。
- 离线或异常设备数。
- 活跃告警数。

卡片应显示简短趋势或状态说明，例如“在线率 86%”“较昨日新增 2 台”。

5. 筛选工具条

工具条包含：

- 关键词搜索：匹配设备名称或设备 ID。
- 设备类型筛选：传感器、摄像头、执行器、控制器。
- 协议筛选：MQTT、HTTP、Modbus TCP。
- 状态筛选：在线、离线、断开、错误。
- 重置按钮。

筛选区保持单行优先，窄屏时换行。

6. 设备表格

表格为主工作区，字段包括：

- 设备信息：名称、设备 ID。
- 类型。
- 协议。
- 状态。
- 最近更新时间。
- 操作：查看、编辑、删除。

交互要求：

- 点击表格行选中设备，并更新右侧详情侧栏。
- 选中行有明确背景色和左侧强调线。
- 状态用文本加颜色标签展示，不能只依赖颜色。
- 操作按钮有 hover、focus-visible 和禁用状态。

7. 详情侧栏

右侧详情侧栏展示当前选中的设备。如果没有选择设备，则显示引导空状态。

内容包括：

- 设备名称、设备 ID、类型和协议。
- 当前状态与最后更新时间。
- 配置摘要，例如主机、端口、采样间隔或标签数量。
- 传感器数量或摄像头流状态的摘要。
- 最近告警摘要；没有告警时显示明确空状态。
- 快捷操作：编辑配置、更新状态、删除设备。

详情侧栏不替代编辑弹窗；它用于快速查看和上下文操作。

8. 新增/编辑设备弹窗

保留当前新增和编辑能力，但重做样式：

- 使用居中 Modal。
- 表单分组为基础信息和通讯配置。
- 输入框、下拉框、按钮使用统一圆角、边框、焦点态。
- 校验错误显示在字段附近。
- 保存中显示 loading 状态，避免重复提交。

## 视觉规范

色彩：

- 深色框架背景：`#0f172a`。
- 侧边栏背景：`#172033`。
- 主内容背景：`#eef3f8`。
- 卡片背景：`#ffffff`。
- 主色：`#2563eb`。
- 成功色：`#16a34a`。
- 警告色：`#f59e0b`。
- 危险色：`#dc2626`。
- 正文色：`#172033`。
- 次要文字：`#64748b`。
- 边框色：`#dbe3ee`。

字体：

- 使用现有页面的 Fira Sans / Fira Code 字体组合。
- 标题使用较高字重，正文保持清晰可读。
- 数字指标可以使用等宽字体增强数据感。

间距和形状：

- 页面主体使用 16-24px 间距。
- 卡片圆角 14-16px。
- 按钮和输入框圆角 8-10px。
- 卡片阴影要轻，避免过度玻璃拟态或大屏炫光。

图标：

- 使用统一 SVG 图标。
- 不使用 emoji 作为功能图标。
- 图标尺寸统一为 16px 或 20px。

## 交互行为

数据加载：

- 页面加载时请求设备列表。
- 加载中显示骨架屏或轻量 loading。
- 加载失败时显示错误卡片和重试按钮。

筛选：

- 本地筛选优先复用已有数据。
- 搜索关键词实时生效，但应保持输入响应流畅。
- 重置按钮清空所有筛选条件。

选择设备：

- 默认选中列表第一项。
- 用户点击某行后，右侧详情侧栏更新。
- 删除当前选中设备后，自动选择剩余列表第一项；如果没有设备，显示空状态。

新增设备：

- 点击“新增设备”打开弹窗。
- 保存成功后关闭弹窗、刷新列表并选中新设备。
- 保存失败时在弹窗内显示错误。

编辑设备：

- 点击编辑打开同一个弹窗，带入当前数据。
- 保存成功后更新表格与详情侧栏。

删除设备：

- 删除前二次确认。
- 删除成功后更新列表、统计卡和详情侧栏。

响应式：

- 桌面端保留左侧导航、主表格和右侧详情栏。
- 平板端详情侧栏可下移到表格下方。
- 手机端表格切换为卡片列表，侧边导航收敛。

可访问性：

- 所有按钮和交互元素支持键盘 focus-visible。
- 图标按钮提供文本标签或 `aria-label`。
- 状态标签包含文字，不只用颜色区分。
- 动效尊重 `prefers-reduced-motion`。

## 实现约束

- 继续在 `frontend/index.html` 中实现。
- 使用原生 HTML、CSS 和 JavaScript。
- 不修改设备管理后端 API。
- 不提交 `.superpowers/` 可视化辅助临时文件。
- 如果发现 `.superpowers/` 未被忽略，应建议加入 `.gitignore`，但本次设计文档不负责修改。

## 测试与验收

验收标准：

- 页面视觉符合混合型运营后台风格。
- 当前设备管理功能仍可用：列表、搜索筛选、新增、编辑、删除、状态展示。
- 统计卡随设备数据变化更新。
- 点击设备行能更新详情侧栏。
- 没有 emoji 功能图标。
- 桌面和移动端无明显布局溢出。
- 关键按钮有 hover 和键盘焦点态。

建议验证：

- 打开静态页面检查初始布局。
- 使用演示 API 或当前后端服务验证设备列表加载。
- 手动测试新增、编辑、删除和筛选。
- 在 375px、768px、1024px、1440px 宽度检查响应式布局。
